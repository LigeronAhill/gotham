# Gotham

**Gotham** — это генератор веб-приложений на Go, использующий современный стек технологий:

- **Go** (1.26+) — высокопроизводительный бэкенд  
- **Templ** — типобезопасные HTML-шаблоны  
- **HTMX** — динамический интерфейс без написания JavaScript  
- **AlpineJS** — лёгкий реактивный слой на клиенте  
- **TailwindCSS** — утилитарный CSS-фреймворк  
- **PostgreSQL** — основная база данных (через pgxpool)  

Проект создаётся с готовой структурой, настройкой `justfile` для быстрых команд, поддержкой горячей перезагрузки и автоматической генерацией кода Templ.

---

## Особенности

- Генерация готового каркаса приложения с чистой архитектурой.
- Интерактивный мастер для настройки имени, модуля, уровня логирования, хоста, порта и строки подключения к БД.
- Автоматическая инициализация Git-репозитория.
- Поддержка горячей перезагрузки через `just watch` (Tailwind + Templ одновременно).
- Встроенная поддержка **just** для упрощения повседневных задач.

## Установка генератора

Установите **Gotham** одной командой:

```bash
go install github.com/LigeronAhill/gotham@latest
```

Убедитесь, что `$GOPATH/bin` (обычно `~/go/bin`) добавлен в `PATH`. После этого команда `gotham` будет доступна в терминале.

## Создание нового проекта

Перейдите в директорию, где хотите разместить проект, и выполните:

```bash
gotham
```

Запустится интерактивный мастер, который запросит:

- **Application title** — название проекта (например, `MyApp`).  
  Из него формируется имя папки (в нижнем регистре, пробелы заменяются на `_`).
- **Module path** — путь модуля Go (например, `github.com/username/myapp`).
- **Log level** — уровень логирования (`debug`, `info`, `warn`, `error`).
- **Postgres URL** — строка подключения к БД (по умолчанию `postgres://postgres:postgres@localhost:5432/gotham`).
- **Application host** — хост для сервера (по умолчанию `127.0.0.1`).
- **Application port** — порт (по умолчанию `3000`).
- **Admin password for pprof** — пароль для доступа к профилировщику (pprof).

После ответов на все вопросы генератор:

1. Создаёт папку проекта.
2. Копирует все файлы шаблона (с переименованием `env.template` → `.env`, `gitignore.template` → `.gitignore` и заменой плейсхолдеров).
3. Выполняет `go mod init` с указанным модульным путём.
4. Запускает `go mod tidy`.
5. Устанавливает зависимости Node.js через `npm install`.
6. Устанавливает **Templ** и генерирует код.
7. Инициализирует Git-репозиторий.

По окончании вы увидите:

```
✅ Project initialized successfully. Good luck with development!
```

## Структура проекта (кратко)

```
myapp/
├── cmd/
│   └── app/               # точка входа (main.go)
├── internal/              # внутренние пакеты
│   ├── config/            # загрузка конфигурации
│   ├── database/          # подключение к БД (pgxpool)
│   ├── handlers/          # HTTP-обработчики
│   ├── logger/            # настройка логирования
│   ├── middleware/        # промежуточные обработчики
│   ├── models/            # структуры данных
│   ├── repositories/      # работа с БД
│   ├── router/            # маршрутизация
│   ├── server/            # настройка HTTP-сервера
│   ├── services/          # бизнес-логика
│   └── utils/             # вспомогательные функции
├── public/                # статические файлы (CSS, JS, изображения)
│   ├── assets/
│   ├── fonts/
│   └── icons/
├── ui/                    # Templ-компоненты
│   ├── components/        # переиспользуемые компоненты
│   └── layouts/           # макеты страниц
├── .env                   # переменные окружения (создаётся из env.template)
├── .gitignore             # игнорируемые файлы
├── input.css              # входной файл для Tailwind
├── justfile               # команды для just
├── package.json           # зависимости Node.js
├── go.mod                 # зависимости Go
└── go.sum
```

## Команды just

Перейдите в папку проекта и выполните `just --list`, чтобы увидеть все доступные команды:

| Команда | Описание |
|---------|----------|
| `just run-app` | Запустить приложение (с автоматической генерацией Templ) |
| `just watch` | **Горячая перезагрузка** – параллельно запускает `watch-style` и `watch-templ` |
| `just gen-templ` | Сгенерировать код Templ (один раз) |
| `just watch-templ` | Следить за изменениями `.templ` и перезапускать сервер с прокси-адресом |
| `just gen-style` | Собрать CSS (один раз) |
| `just watch-style` | Следить за изменениями `input.css` и пересобирать TailwindCSS |
| `just fmt` | Форматировать Go-код |
| `just vet` | Запустить `go vet` |
| `just lint` | Запустить `golangci-lint` (требуется установленный линтер) |
| `just test` | Запустить тесты с покрытием |
| `just test-cover` | Запустить тесты и открыть HTML-отчёт о покрытии |
| `just check` | Выполнить `fmt`, `vet`, `test`, `lint` (полная проверка) |
| `just install-deps` | Установить зависимости (TailwindCSS, Templ и обновить Go-модули) |

## Разработка с горячей перезагрузкой

Для удобной разработки используйте:

```bash
just watch
```

Эта команда запускает параллельно:

- **TailwindCSS CLI** в режиме `--watch` для автоматической пересборки CSS.
- **Templ** в режиме `--watch` с прокси-сервером, который перезапускает приложение при изменении `.templ` или `.go` файлов.

Теперь вы можете редактировать код и сразу видеть изменения в браузере без ручного перезапуска.
